### 1. AI提示词优化
**文件**：`client/src/prompts/contractAnalysisPrompt.ts`（第148-205行）

#### 新增"公司业务分类智能判断"章节

```markdown
#### 九、公司业务分类智能判断（基于系统配置标准）
18. **公司业务分类**（businessClassification）：根据工程实际情况智能判断业务分类档次

**业务分类标准**：
- **鼓励类**（低风险，优先承保）：
  * 工程造价 < 5000万元
  * 工程分类1-2类（普通基建/装饰/机电能源）
  * 风险等级1-2级
  * 无特殊风险因素（无爆破、隧道、深基坑等）
  * 常见类型：住宅小区、室内装修、绿化、市政道路（无桥隧）

- **一般类**（中等风险，正常承保）：
  * 工程造价 5000万-2亿元
  * 工程分类2-3类
  * 风险等级3级
  * 常规施工方法，无特殊工艺
  * 常见类型：普通市政、道路、轻型基础设施

- **谨慎类**（中高风险，谨慎评估）：
  * 工程造价 2亿-5亿元
  * 工程分类3类
  * 风险等级4级
  * 有一定难度（深基坑<10米、市政桥梁、小型隧道）
  * 常见类型：市政桥梁、小型隧道、机电安装

- **限制类**（高风险，严格限制）：
  * 工程造价 5亿-10亿元
  * 工程分类4类
  * 风险等级4-5级
  * 高难度施工（爆破、隧道占比高、大型钢结构、幕墙）
  * 常见类型：大型隧道、高架桥、水电能源工程

- **严格限制类**（极高风险，原则上不承保）：
  * 工程造价 > 10亿元
  * 风险等级5级
  * 极高风险作业（纯隧道、纯桥梁、人工拆除、无资质施工）
  * 特殊高风险项目

**判断流程**：
1. 根据工程造价初步定位分类档次
2. 根据工程分类调整（高风险工程→上调档次）
3. 根据风险等级确认（风险等级高→上调档次）
4. 根据特殊工艺微调（有爆破、隧道→上调档次）
5. 综合评估后给出最终分类、置信度和详细理由

**输出格式**：
{
  "category_level": "鼓励类|一般类|谨慎类|限制类|严格限制类",
  "category_name": "XX类业务",
  "confidence": "high|medium|low",
  "reasoning": "详细说明判断依据：造价XX万（区间），工程分类X类（工程内容），风险等级X级（风险因素），特殊工艺（如有列举），综合判断为XX类业务"
}
```

**关键改进**：
- ✅ 明确5个分类的判断标准（造价、工程分类、风险等级）
- ✅ 提供清晰的5步判断流程
- ✅ 要求输出置信度和详细理由
- ✅ 强调基于合同实际内容，不套用模板

## 📊 AI判断逻辑

### 多维度综合判断

AI不再只看项目名称，而是综合多个维度：

1. **工程造价**（主要维度）
   - <5000万 → 倾向鼓励类
   - 5000万-2亿 → 倾向一般类
   - 2亿-5亿 → 倾向谨慎类
   - 5亿-10亿 → 倾向限制类
   - >10亿 → 倾向严格限制类

2. **工程分类**（调整维度）
   - 1类（普通基建/装饰）→ 可能降低档次
   - 2类（机电/能源）→ 保持档次
   - 3类（高风险/农村）→ 可能提升档次
   - 4类（极高风险）→ 明显提升档次

3. **风险等级**（确认维度）
   - 1-2级 → 低风险业务
   - 3级 → 中等风险业务
   - 4-5级 → 高风险业务

4. **特殊工艺**（微调维度）
   - 爆破、隧道、深基坑 → 上调档次
   - 常规施工 → 保持档次
