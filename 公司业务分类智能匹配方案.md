# 公司业务分类智能匹配方案

## 📊 当前情况分析

### 现状
1. **后端已有匹配功能**：`configService.ts` 中的 `matchBusinessClassification` 方法
2. **匹配方式**：基于项目名称的关键词匹配（48个关键词+同义词+加权评分）
3. **问题**：之前因匹配不准确，已移除自动匹配，改为手动选择
4. **用户反馈**："工程分类以及业务分类判断错误，所以合同解析出来的内容都必须依据实际情况，不能随意揣测"

### 核心问题
1. **识别率低**：仅基于项目名称，很多项目无法匹配
2. **准确率低**：单一维度匹配，容易误判
3. **缺乏上下文**：没有考虑造价、工程分类、特殊工艺等因素
4. **用户不信任**：之前错误匹配导致用户不信任自动功能

---

## 🎯 方案对比

### 方案A：优化规则匹配（快速，但仍有限）

**核心思路**：改进现有的关键词匹配算法

#### 优化点：
1. **多维度匹配**：不仅看项目名称，还要结合：
   - 工程造价（造价越低越倾向鼓励类）
   - 工程分类（1-4类）
   - 特殊工艺（是否有爆破、隧道等）
   - 施工合同类型（总包 vs 劳务分包）

2. **增加负向关键词**：
   - 看到"爆破"、"隧道"→ 提高风险等级
   - 看到"住宅"、"装修"→ 降低风险等级

3. **提高阈值**：
   - 当前：30分就算匹配
   - 优化：只有60分以上才算高置信度匹配

**优点**：实现快，不需要AI
**缺点**：仍然基于规则，灵活性差，准确率提升有限

---

### 方案B：AI智能匹配（推荐⭐⭐⭐⭐⭐）

**核心思路**：利用AI的语义理解能力，让AI根据合同完整内容判断业务分类

#### 实现方式：

**1. 在合同分析时直接输出业务分类**

修改AI提示词，增加业务分类判断逻辑：

```markdown
### 九、业务分类判断（新增）
17. **公司业务分类**（businessClassification）：根据以下标准判断
   - **鼓励类**（1-2级风险）：
     * 工程造价 < 5000万
     * 常规房建、装修、绿化
     * 无特殊风险因素
     * 工程分类1-2类

   - **一般类**（3级风险）：
     * 工程造价 5000万-2亿
     * 普通市政、道路
     * 常规施工方法
     * 工程分类2-3类

   - **谨慎类**（4级风险）：
     * 工程造价 2亿-5亿
     * 有深基坑（<10米）
     * 市政桥梁、小型隧道
     * 工程分类3类

   - **限制类**（5级风险）：
     * 工程造价 5亿-10亿
     * 有爆破、隧道、桥梁占比高
     * 大型钢结构、幕墙
     * 工程分类4类

   - **严格限制类**（极高风险）：
     * 工程造价 > 10亿
     * 纯隧道、纯桥梁
     * 人工拆除、无资质施工
     * 特殊高风险作业

   **判断依据**：
   1. 工程规模（造价、面积、工期）
   2. 工程分类（1-4类）
   3. 风险等级（1-5级）
   4. 特殊工艺（爆破、隧道、深基坑等）
   5. 施工合同类型（总包 vs 劳务分包）

   **输出格式**：
   {
     "category_level": "鼓励类|一般类|谨慎类|限制类|严格限制类",
     "category_name": "XX类业务",
     "confidence": "high|medium|low",
     "reasoning": "判断理由说明"
   }
```

**2. AI返回分类+置信度+理由**

```json
{
  "businessClassification": {
    "category_level": "鼓励类",
    "category_name": "鼓励类业务",
    "confidence": "high",
    "reasoning": "工程造价1200万（<5000万），工程为普通住宅小区建设（1类工程），无特殊风险因素，风险等级1级，符合鼓励类标准"
  }
}
```

**3. 前端显示AI推荐**

```typescript
// 在ContractAnalysis.tsx中
{
  key: 'businessClassification',
  label: '公司业务分类',
  value: extractedData.businessClassification?.category_name || '请选择',
  highlight: !!extractedData.businessClassification,
  // 显示AI推荐标记
  aiRecommended: !!extractedData.businessClassification,
  confidence: extractedData.businessClassification?.confidence,
  reasoning: extractedData.businessClassification?.reasoning
}
```

**4. 用户可以看到：**
- ✅ AI推荐的分类
- ✅ 推荐置信度（高/中/低）
- ✅ 推荐理由（为什么推荐这个分类）
- ✅ 可以接受或修改

**优点**：
- ✅ 利用AI的语义理解，匹配更准确
- ✅ 考虑了合同的完整上下文
- ✅ 提供推荐理由，用户可验证
- ✅ 显示置信度，用户可评估可信度
- ✅ 保留用户修改权利

**缺点**：
- 需要修改AI提示词
- 依赖AI准确性

### 方案C：混合智能匹配（最佳⭐⭐⭐⭐⭐）

**核心思路**：规则匹配 + AI推荐 + 用户确认

#### 三层保障：

**第一层：规则快速匹配（后端）**
- 如果项目名称明确匹配（>80分），直接推荐
- 适用于常见项目类型

**第二层：AI智能分析（前端）**
- 如果规则匹配不确定，调用AI分析
- AI根据合同完整内容判断
- 返回分类+置信度+理由

**第三层：用户确认（前端）**
- 显示推荐结果
- 显示推荐理由
- 显示置信度
- 用户可以接受或修改

#### 实现流程：

```typescript
// 1. 合同解析完成
const analysisResult = await analyzeContract(contractText);

// 2. 规则匹配（后端API）
const ruleMatch = await matchByRules(analysisResult.projectName);

// 3. 如果规则匹配置信度高
if (ruleMatch && ruleMatch.confidence === 'high') {
  analysisResult.businessClassification = ruleMatch;
}
// 4. 如果规则匹配不确定，让AI判断
else {
  // AI已经在合同分析时输出了业务分类
  // analysisResult.businessClassification 已经有值
}

// 5. 前端显示
{
  businessClassification: {
    category_name: "鼓励类业务",
    confidence: "high", // 或 medium, low
    reasoning: "工程造价1200万，工程为普通住宅...",
    matchMethod: "规则匹配" // 或 "AI分析"
  }
}
```

#### 前端界面：

```tsx
// AI推荐卡片
{extractedData.businessClassification && (
  <div className="ai-recommendation-card">
    <Tag color={getConfidenceColor(extractedData.businessClassification.confidence)}>
      AI推荐: {extractedData.businessClassification.category_name}
    </Tag>
    <div className="confidence">
      置信度: {extractedData.businessClassification.confidence === 'high' ? '高' :
               extractedData.businessClassification.confidence === 'medium' ? '中' : '低'}
    </div>
    <div className="reasoning">
      理由: {extractedData.businessClassification.reasoning}
    </div>
    <div className="match-method">
      匹配方式: {extractedData.businessClassification.matchMethod}
    </div>
  </div>
)}

// 用户仍可修改
<Select
  value={extractedData.businessClassification?.category_name}
  onChange={handleUserModify}
>
  {businessClassifications.map(...)}
</Select>
```
