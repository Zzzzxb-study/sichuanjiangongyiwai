# 智能业务分类推荐系统

## 方案概述

基于AI分析合同的关键特征，智能推荐业务分类，提高自动化程度和准确性。

## 核心思路

### 1. 多维度特征提取
AI从合同中提取以下关键信息：
- **工程规模**: 造价、面积、工期
- **技术特征**: 是否有爆破、深基坑、隧道、高空作业等
- **工程类型**: 房建、市政、水利、交通等
- **特殊条款**: 保险要求、风险提示
- **地理位置**: 地区、气候、地质条件

### 2. 智能推荐规则

#### 鼓励类（低风险）：
- 工程造价 < 5000万
- 常规房建、装修、绿化
- 无特殊风险因素
- 地质条件良好

#### 一般类（中等风险）：
- 工程造价 5000万-2亿
- 普通市政、道路
- 轻型基础设施
- 常规施工方法

#### 谨慎类（中高风险）：
- 工程造价 2亿-5亿
- 有深基坑（<10米）
- 市政桥梁、小型隧道
- 机电安装工程

#### 限制类（高风险）：
- 工程造价 5亿-10亿
- 有爆破、隧道、桥梁占比高
- 大型钢结构、幕墙
- 水电、能源工程

#### 严格限制类（极高风险）：
- 工程造价 > 10亿
- 纯隧道、纯桥梁
- 人工拆除、无资质施工
- 特殊高风险作业

### 3. 实现方案

#### 后端新增API：
```typescript
// 智能推荐API
POST /api/config/recommend-classification
{
  contractText: string,
  extractedData: {
    projectName: string,
    projectCost: number,
    buildingArea: number,
    specialFeatures: string[],
    location: string,
    workerCount: number,
    duration: number
  }
}

// 返回
{
  recommended_category: string,
  confidence: 'high' | 'medium' | 'low',
  reasoning: string,
  key_factors: {...},
  risk_assessment: string,
  alternative_categories: [...]  // 备选方案
}
```

### 4. AI Prompt优化

```
你是一位专业的建筑工程保险风险评估专家。

请分析以下合同信息，推荐合适的业务分类：

## 分析维度
1. 工程规模（造价、面积、工期）
2. 技术复杂度（是否有特殊施工工艺）
3. 风险因素（地质、气候、周边环境）
4. 施工难度（高空、爆破、深水等）
5. 历史数据（类似工程的赔付率）

## 业务分类标准
- 鼓励类：低风险，优先承保（<5000万，常规工程）
- 一般类：中等风险，正常承保（5000万-2亿，标准工程）
- 谨慎类：中高风险，谨慎评估（2亿-5亿，有一定难度）
- 限制类：高风险，严格限制（5亿-10亿，高难度）
- 严格限制类：极高风险，原则上不承保（>10亿，极高风险）

## 输出格式
{
  "recommended_category": "鼓励类|一般类|谨慎类|限制类|严格限制类",
  "confidence": "high|medium|low",
  "reasoning": "详细推荐理由",
  "key_factors": {
    "project_cost": number,
    "special_features": string[],
    "risk_level": 1-5,
    "complexity": "low|medium|high"
  },
  "risk_assessment": "风险评估描述",
  "alternatives": ["备选分类1", "备选分类2"]
}

请基于实际工程数据和风险评估，给出专业推荐。
```

## 优势

1. **保持自动化**：AI自动分析并推荐
2. **提高准确性**：基于实际合同数据，不是简单关键词匹配
3. **透明可解释**：显示推荐理由，用户可验证
4. **灵活可调**：用户可接受或修改推荐
5. **持续学习**：可基于历史数据优化

## 实施建议

### 阶段1：基于规则的推荐（快速实现）
- 硬编码业务规则
- 基于提取的数值判断
- 无需额外AI调用

### 阶段2：AI增强推荐（最佳）
- 利用现有AI分析
- 在合同解析时同时输出分类推荐
- 提供详细理由

### 阶段3：数据驱动优化（长期）
- 收集用户反馈
- 优化推荐算法
- 提高准确率

## 关键点

✅ **不要完全移除自动推荐** - 降低实用性
✅ **提供推荐理由** - 增加透明度
✅ **允许用户调整** - 保持人工判断
✅ **显示置信度** - 评估可靠性
✅ **提供备选方案** - 给予选择余地
