## 🎯 需求
在项目基本信息中加入"施工合同类型"字段，用于标识合同的类型（总承包合同、专业分包合同、劳务分包合同等）。

## ✅ 完成的修改

### 1. 数据接口定义
**文件**：`client/src/services/aiService.ts`（第43-52行）

添加了`contractType`字段到`ContractAnalysisResult`接口：

```typescript
/**
 * 施工合同类型
 * - 总承包合同：业主与总承包方签订的合同
 * - 专业分包合同：总承包方与专业分包方签订的合同（如幕墙、消防、电梯等）
 * - 劳务分包合同：总承包方与劳务分包方签订的合同（纯劳务）
 * - 施工总承包合同：与总承包合同类似
 * - 工程分包合同：一般性工程分包
 * - 其他：未明确或特殊类型
 */
contractType: string;
```

#### 2.1 添加提取规则（第53-64行）

```markdown
3. **施工合同类型**（contractType）：根据合同标题、签约双方和内容判断合同类型
   - **总承包合同**：业主（发包人）与总承包方签订，包含整体工程
   - **专业分包合同**：总承包方作为发包方，与专业分包方签订特定专业工程（如幕墙、消防、电梯、钢结构等）
   - **劳务分包合同**：只提供劳务服务，不提供材料设备，通常标注"劳务分包"或"劳务承包"
   - **施工总承包合同**：与总承包合同类似，业主与施工总承包方签订
   - **工程分包合同**：一般性工程分包，包含部分工程内容
   - **其他**：合同中未明确类型或特殊类型
   - **判断依据**：
     * 查看合同标题（如"XX工程施工总承包合同"、"XX专业分包合同"）
     * 查看发包人（如果是建筑公司而非业主，可能是分包合同）
     * 查看承包范围（整体工程=总承包，特定专业=专业分包）
   - 如果合同未明确类型，填写："未明确"
```

**关键点**：
- 提供了6种合同类型的明确定义
- 给出了3个判断依据（标题、发包人、承包范围）
- 明确未明确时的处理方式

## 📊 字段详情

### 施工合同类型选项及含义

| 选项 | 说明 | 典型特征 |
|------|------|----------|
| **总承包合同** | 业主与总承包方签订 | - 发包人是业主/建设单位<br>- 承包范围：整体工程<br>- 合同标题："XX工程施工总承包合同" |
| **施工总承包合同** | 业主与施工总承包方签订 | - 发包人是业主/建设单位<br>- 承包范围：整体工程施工<br>- 与总承包合同类似 |
| **专业分包合同** | 总承包方与专业分包方签订 | - 发包人是建筑公司（非业主）<br>- 承包范围：特定专业（幕墙、消防、电梯、钢结构等）<br>- 合同标题："XX专业分包合同" |
| **工程分包合同** | 总承包方与分包方签订 | - 发包人是建筑公司<br>- 承包范围：部分工程内容<br>- 非特定专业 |
| **劳务分包合同** | 总承包方与劳务分包方签订 | - 只提供劳务服务<br>- 不提供材料设备<br>- 合同标题标注"劳务分包" |
| **未明确** | 合同未明确类型 | - 合同标题不明确<br>- 签约双方关系不清晰<br>- 承包范围模糊 |

## 📝 AI判断逻辑

### 三大判断依据：

1. **合同标题**（最直接）
   - 包含"总承包" → 总承包合同
   - 包含"专业分包" + 专业名称 → 专业分包合同
   - 包含"劳务分包" → 劳务分包合同

2. **发包人身份**（关键线索）
   - 业主/建设单位/开发商 → 总承包合同
   - 建筑公司/总承包方 → 分包合同

3. **承包范围**（辅助判断）
   - 整体工程/全部工程 → 总承包合同
   - 特定专业（幕墙、消防等）→ 专业分包合同
   - 纯劳务服务 → 劳务分包合同
