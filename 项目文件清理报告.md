## 🛠️ 清理方案

### 方案一：仅从 Git 移除（推荐）
保留本地文件，但从 Git 仓库中移除，不影响本地开发。

```bash
# 执行清理脚本
cd "E:\软件开发\四川地区建工意外"

# 从 Git 移除文件（保留本地）
git rm --cached test_match.js
git rm --cached frontend_tests.py
git rm --cached excel_structure.json
git rm --cached nul
git rm --cached "AI提示词模型优化问题修复说明.md"
git rm --cached "PDF智能提取优化实施说明.md"
git rm --cached "PDF智能提取优化方案.md"
git rm --cached "UI重设计需求.txt"
# ... 其他文件

# 移除文档文件夹
git rm -r --cached "条款及费率文件"
git rm -r --cached "费率规则解析文件"

# 移除 server 测试文件
git rm --cached server/check-business-classifications.js
git rm --cached server/check-business-classifications2.js
git rm --cached server/check-tables.js
git rm --cached server/database.db
git rm --cached server/nul
git rm -r --cached server/scripts

# 移除 client 测试文件
git rm --cached client/public/test-api.html

# 提交更改
git commit -m "chore: 移除开发文档和测试文件"
```

### 方案二：完全删除文件
如果确定不需要这些文件，可以直接删除。

```bash
# 删除根目录测试文件
rm test_match.js frontend_tests.py excel_structure.json nul

# 删除开发文档
rm *.md *.txt
# 保留 README.md 和重要的配置文档

# 删除文档文件夹
rm -rf "条款及费率文件" "费率规则解析文件"

# 删除 server 测试文件
rm server/check-*.js server/nul server/database.db
rm -rf server/scripts

# 删除 client 测试文件
rm client/public/test-api.html

# 提交更改
git add .
git commit -m "chore: 清理开发文档和测试文件"
```

## 🎯 执行建议

### 推荐执行步骤：

1. **先备份重要文档**
   ```bash

   mkdir docs_backup
   cp *.md docs_backup/
   cp *.txt docs_backup/
   cp -r 条款及费率文件 docs_backup/
   cp -r 费率规则解析文件 docs_backup/
   ```

2. **更新 .gitignore**
   先更新 `.gitignore` 文件，防止以后误提交

3. **从 Git 移除文件**
   使用 `git rm --cached` 从 Git 移除但保留本地文件

4. **测试应用**
   确保删除这些文件后应用仍能正常运行

5. **提交更改**
   ```bash
   git add .

   ```
