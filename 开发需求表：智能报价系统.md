四川建工意外险智能报价系统开发需求表

## **1\. 核心计算引擎 (Calculation Engine)**

| 需求编号 | 功能模块 | 详细描述 | 逻辑参考 |
| :---- | :---- | :---- | :---- |
| **CE-01** | **主险计算模块** | 实现 $P\_{main} \= B \\times 费率 \\times (保额/10万) \\times \\prod K\_{n}$。 |  |
| **CE-02** | **线性插值算法** | 封装通用函数 Calc\_Interpolation，处理造价、面积、工期的非节点费率计算。 |  |
| **CE-03** | **附加险联动** | 实现医疗(M系数)、津贴(天数插值)计算；高原病需基于“主险+其他附加险”总额的 8.1% 计算。 |  |
| **CE-04** | **路径自动分流** | 根据“项目性质”自动切换：非农村（按造价/K1）或农村（按面积/K2/锁定三类工程）。 |  |

## **2\. AI 合同解析模块 (OCR & LLM)**

| 需求编号 | 功能模块 | 详细描述 | 约束条件 |
| :---- | :---- | :---- | :---- |
| **AI-01** | **关键因子提取** | 从合同中提取：项目名称、造价、开竣工日期、工程地址、施工单位、资质。 | 需支持 PDF/Word。 |
| **AI-02** | **工程分类判定** | LLM 根据合同内容关键词匹配“一至四类”工程分类。 | 允许人工手动修正。 |
| **AI-03** | **风险地标识别** | 通过地址判断是否涉及“高海拔/高原”地区，自动预勾选高原病条款。 |  |

## **3\. 历史承保数据库 (Historical Data)**

| 需求编号 | 功能模块 | 详细描述 | 状态 |
| :---- | :---- | :---- | :---- |
| **DB-01** | **数据归一化存储** | 存储拆分后的基准费率（10万主险/1万医疗/10元津贴），作为报价基准。 | 已完成清洗逻辑。 |
| **DB-02** | **智能费率推荐** | 基于当前项目维度（造价、工期、类型）匹配历史相似件，计算平均费率。 |  |
| **DB-03** | **区间合规监控** | 实时显示历史同类项目的费率上限 (Max) 与下限 (Min)。 |  |

## **4\. 前端交互与功能 (UI/UX)**

| 需求编号 | 功能模块 | 详细描述 | 备注 |
| :---- | :---- | :---- | :---- |
| **UI-01** | **因子确认看板** | 左右分栏结构：左侧合同预览，右侧 AI 提取因子卡片。 | 关键字段标红提醒。 |
| **UI-02** | **参数调节面板** | 支持手动输入 K7(管理水平)、K8(经验赔付)，并限制在费率表允许范围内。 |  |
| **UI-03** | **报价对比视图** | 同屏显示：1. 精算底线值；2. 历史平均值；3. 最终报价建议值。 |  |
| **UI-04** | **报价单导出** | 一键生成 PDF 报价单，包含费率维度快照和分项保费清单。 |  |

## **5\. 系统配置与安全 (System & Security)**

| 需求编号 | 功能模块 | 详细描述 | 逻辑参考 |
| :---- | :---- | :---- | :---- |
| **SYS-01** | **业务分类配置表** | 支持管理后台动态修改一至四类工程的具体包含项，无需修改代码。 |  |
| **SYS-02** | **费率节点维护** | 维护 K/M/Q 系数的数值节点，方便年度费率调整。 |  |
