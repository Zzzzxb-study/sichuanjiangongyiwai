# 附加意外医疗险保费计算规则 PRD (V1.0)

- **功能名**：附加意外伤害医疗保险计算引擎
- **需求描述**：
在主险基础上，支持根据医疗保额、免赔额、给付比例及社保情况等参数，动态计算附加医疗险保费。支持造价/面积双路径分流及全参数线性插值。
- **概述**：
附加医疗险是建工险的重要组成部分。本模块旨在通过高精度的参数调节（M1-M5）与工程风险调节（K1-K7），实现针对不同医疗保障方案的精准定价。

### 实现逻辑 (Calculation Logic)

#### 1. 总公式
$$Premium_{medical} = BasePremium \times \prod ParameterAdjustment (M_{1-5}) \times \prod RateAdjustment (K_{1-7})$$

#### 2. 基准保费 ($BasePremium$) 判定
* **非农村项目**：$BasePremium = 0.33\text{‰} \times \text{项目总造价}$
* **农村建房项目**：$BasePremium = 0.9 \times \text{项目总面积(㎡)}$

#### 3. 参数调整系数 ($M$ 系列 - 线性插值)
所有医疗参数均需遵循**线性插值算法**：
* **$M_1$ (保额)**：节点 [2k, 5k, 10k, 20k, 30k, 50k, 100k, 200k]。
 * 边界处理：$<2k$ 取 0.58；$>200k$ 取 1.41。
* **$M_2$ (免赔额)**：节点 [0, 100, 200, 300, 400, 500, 1000, 2000]。
 * 边界处理：$>2000$ 取 0.72。
* **$M_3$ (给付比例)**：节点 [50%, 60%, 70%, 80%, 90%, 100%]。
* **$M_4$ (社保情况)**：映射逻辑（参加=1.0，未参加=1.5）。
* **$M_5$ (第三方补偿)**：映射逻辑（是=0.9，否=1.0）。

#### 4. 费率调整系数 ($K$ 系列 - 工程相关)
* **$K_1$ (造价/面积)**：非农村按造价插值；农村按面积插值。*（注意：附加险的插值节点与主险不同，需独立建模）*
* **$K_3$ (合同类型)**：联动“工程分类”，若为劳务分包，按一至四类分别取 [4, 5, 6, 7]。
* **$K_4$ (工程类型)**：用户在 [0.8-2.5] 的对应等级区间内手动输入。
* **$K_5$ (工期)**：按照一年、二年...五年节点执行插值。
* **$K_6$ (资质)**：特级(0.9) / 一级(0.95) / 二级(1.0) / 三级(1.1) / 不分类(1.2)。
* **$K_7$ (风险管理)**：三档区间手动录入。

### 功能细节描述
| 字段名称 | 规则细节 | 异常处理 |
| :--- | :--- | :--- |
| **插值逻辑一致性** | 所有标注“线性插值”的系数，需共用主险的 `linearInterpolation` 函数。 | 确保 $M_1, M_2, M_3, K_1, K_2, K_5$ 逻辑统一。 |
| **农村项目锁定** | 当项目性质为“农村”时，$K_1$ 调整系数失效，强制执行工程面积调整系数逻辑。 | 前端置灰不适用项。 |
| **风险信息缺位** | 若某项调整系数（如施工资质）不确定，默认取值 1.0。 | 系统预设缺省值。 |
| **M1/M2 边界值** | $M_1$ 在 0-2000 之间不插值，固定为 0.58。 | 严格执行注 2 规则。 |

**产品经理总结**：
附加险计算逻辑与主险高度相似，但在**基准计算**和**内部参数（M系数）**上有显著差异。建议开发时将“线性插值”算法抽象为底层服务，并在计算附加险时，先计算 $M$ 系数集合，再乘以后置的工程 $K$ 系数。
