# AI提示词模板化问题修复说明

## 🐛 问题描述

**用户反馈**：
> "我刚刚解析了一个合同，为什么在工程分类以及风险评估中出现'工程包含隧道工程，且桥隧占比超过50%'，合同里面没有相关内容，我感觉每一个解析的合同都是这样的描述"

**根本原因**：
AI将提示词中的**示例**当作了**模板答案**，导致所有合同都返回相同的描述内容。这是大语言模型常见的"过度学习示例"问题。

**问题示例**：
- ❌ 每个合同都填写："工程包含隧道工程，且桥隧占比>50%"
- ❌ 每个合同都填写："工程包含爆破、隧道等特殊施工工艺"
- ✅ 应该根据实际合同内容填写

---

## ✅ 修复方案

### 修改文件：`client/src/prompts/contractAnalysisPrompt.ts`

### 修改1：增强严格解析原则（第8-16行）

#### 新增第5-6条原则：
```markdown
5. **禁止使用示例作为答案**：提示词中的示例仅作参考，必须根据合同实际情况填写
   - ❌ 错误：每个合同都填写"工程包含隧道工程，且桥隧占比>50%"（这是示例，不是答案）
   - ✅ 正确：根据实际合同内容填写，如"合同为普通住宅建筑工程"或"合同中未明确提及"
6. **描述性字段必须真实**：classificationReason、riskReason等描述性字段必须反映合同真实内容
```

**效果**：明确禁止AI套用示例文本作为答案

### 修改2：优化工程分类判断依据说明（第64-69行）

#### 修改前：
```markdown
6. **工程分类判断依据**（classificationReason）：说明选择该分类的理由（如：含有隧道工程且桥隧占比>50%）
```

#### 修改后：
```markdown
6. **工程分类判断依据**（classificationReason）：说明选择该分类的理由
   - ⚠️ **必须基于合同实际内容，严禁使用示例文本**
   - 如果合同是普通房建工程，填写："合同为普通住宅建筑工程"
   - 如果合同是装修工程，填写："合同为室内装修工程"
   - 如果合同有隧道工程，填写："合同包含隧道工程（XX米）"
   - 如果合同未明确工程类型，填写："合同中未明确提及具体工程类型"
```

**效果**：
- 提供**多个不同场景的示例**，避免单一示例被当成模板
- 明确"未明确"情况的处理方式
- 强调"严禁使用示例文本"

### 修改3：优化风险评估说明（第95-100行）

#### 修改前：
```markdown
13. **风险评估说明**（riskReason）：说明风险等级判断依据
```

#### 修改后：
```markdown
13. **风险评估说明**（riskReason）：说明风险等级判断依据
   - ⚠️ **必须基于合同实际内容，严禁编造风险因素**
   - 低风险（1-2级）示例："合同为普通住宅项目，无特殊施工工艺"
   - 中等风险（3级）示例："合同为XX层高层建筑，采用常规施工方法"
   - 高风险（4-5级）示例："合同包含爆破工程、深基坑等特殊施工工艺"
   - 如果合同未提及风险因素，填写："合同中未明确提及特殊风险因素"
```

**效果**：
- 提供不同风险等级的示例
- 明确禁止"编造风险因素"
- 提供"未提及"情况的标准回答

### 修改4：更新JSON模板（第134-162行）

#### 修改前：
```json
{
  "classificationReason": "工程分类判断依据说明",
  "riskReason": "风险评估说明"
}
```

#### 修改后：
```json
{
  "classificationReason": "根据合同实际内容填写，例如：普通住宅建筑工程、市政道路工程等",
  "riskReason": "根据合同实际风险因素填写，如：普通建筑无特殊风险，或：包含深基坑、爆破等高风险作业"
}

**重要提醒**：
- classificationReason 和 riskReason 必须反映合同真实内容
- 如果合同没有隧道，不要写"包含隧道工程"
- 如果合同没有爆破，不要写"包含爆破工程"
- 如果没有特殊风险，写"合同为普通建筑工程，无特殊风险因素"
```

**效果**：
- JSON模板中的占位符更明确
- 在模板末尾添加"重要提醒"再次强调
- 具体说明"什么不要写"

### 修改5：同步更新简化版提示词（第179-213行）

对简化版提示词`CONTRACT_ANALYSIS_PROMPT_SIMPLE`应用相同的修改原则，确保两个版本的提示词都有正确的约束。

## 🔍 修复要点总结

1. **明确禁止套用示例**：在多个位置强调"不要使用示例作为答案"
2. **提供多样化示例**：不再只给隧道示例，而是给出普通住宅、装修、隧道等多种场景
3. **明确"未提及"处理**：告诉AI如果合同中没有相关信息怎么填写
4. **多处重复强调**：在原则、字段说明、JSON模板、重要提醒等多个位置反复强调
5. **简化版同步修复**：确保两个版本的提示词都有正确的约束

## 📝 技术说明

### 为什么会出现这个问题？

大语言模型（LLM）的一个特性是**容易过度学习Few-shot示例**。当提示词中给出示例时：

```
用户的输入：
示例：如：含有隧道工程且桥隧占比>50%

AI的理解：
"用户给了我一个格式模板，我应该按照这个格式回答"
→ 每个合同都输出"含有隧道工程且桥隧占比>50%"
```

### 修复原理：

1. **明确禁止指令**：用⚠️和粗体强调"严禁使用示例文本"
2. **提供多场景示例**：避免单一示例被当成唯一答案
3. **正面+反面示例**：
   - ❌ 错误：每个合同都填写"..."
   - ✅ 正确：根据实际内容填写"..."
4. **多次提醒**：在不同位置重复强调，强化记忆
